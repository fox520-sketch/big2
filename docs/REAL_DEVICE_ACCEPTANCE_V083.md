# v0.8.3 正式上線實機驗收指南

自動測試無法取代真實 Android、iPhone、桌機與多位玩家連線。請上傳 GitHub Pages 後，在首頁的「正式上線實機驗收中心」逐項完成並勾選。

## 測試前準備

1. 確認頁面顯示 v0.8.3。
2. 保留已設定完成的 `src/firebase-config.js`。
3. 確認 Firebase Authentication 匿名登入已啟用。
4. 確認 Firestore Rules 使用免 Cloud Functions 版本。
5. 建議準備 Android 手機、iPhone／iPad、桌機瀏覽器，以及至少兩個不同裝置或無痕視窗。

## PWA 驗收

### Android Chrome
- 安裝遊戲並從桌面圖示啟動。
- 確認以獨立視窗開啟。
- 發布新版後確認更新提示與一鍵修復可用。
- 移除後重新安裝一次。

### iPhone／iPad Safari
- 使用「分享 → 加入主畫面」。
- 從主畫面啟動，確認瀏海與底部安全區沒有遮住按鈕。
- 切換 App、鎖螢幕後返回，確認牌局仍可恢復同步。

### 桌機 Chrome／Edge
- 安裝 PWA、關閉再開啟。
- 確認新版更新與離線首頁正常。

## 多人連續對戰

### 2 真人＋2 AI
- 連續完成 5 局。
- 確認出牌、Pass、AI 行動、下一局、總分與勝場。

### 3 真人＋1 AI
- 連續完成 3 局。
- 輪流鎖螢幕或切換 App，確認不會誤判永久離線。

### 4 真人
- 連續完成 3 局。
- 確認排名使用實際玩家名稱，下一局保留座位及累計分數。

## 斷線、重連與房主轉移

1. 讓非房主斷網 1～2 分鐘，再恢復網路。
2. 確認 AI 接管及真人返回後的座位狀態。
3. 讓房主離線，確認下一位真人接任房主。
4. 確認 AI 回合不會因原房主離線而停住。
5. 使用「重建房間同步」測試 listener 手動恢復。

## 手機 UI

請測試 320、360、390、430px 寬度或相近實機：
- 最左、最右手牌都能完整滑入畫面。
- 選中牌上緣與勾選標記不被裁切。
- 操作面板不遮住手牌。
- 房間密碼鍵盤開啟時輸入欄與加入按鈕可見。
- 直向與橫向都能操作。

## Firebase 讀寫與 listener

開啟「錯誤紀錄中心」：
- 房間中應顯示 1 個 active listener。
- 離開房間後應顯示 0 個 active listener。
- 監聽啟動數減停止數不應長期大於 1。
- 反覆切換前景／背景後，不應持續增加 active listener。
- 房間列表 15 秒內重按應使用快取。

## 發現問題時

1. 先不要重複連點。
2. 在「錯誤紀錄中心」下載錯誤紀錄。
3. 在「多人同步偵錯面板」下載完整診斷檔。
4. 記錄裝置、瀏覽器、操作步驟及發生時間。
