# v0.7.3 手機 UI 檢查清單

## 已優化

- 手機寬度下，牌桌改為單欄排列。
- 操作區在手機上改為底部 sticky，降低出牌/Pass 操作距離。
- 手牌與上一手支援橫向滑動，避免小螢幕擠壓到看不清楚。
- 按鈕高度加大，降低誤觸。
- input/select/button 使用 16px 以上字級，避免 iOS 自動放大。
- 結果表格可橫向捲動。
- 支援 `prefers-reduced-motion`，使用者關閉動畫時會降低動態效果。

## 測試建議

請至少用以下尺寸測試：

- iPhone SE / 375px 寬
- 一般 Android / 390px 寬
- 平板 / 768px 寬
- 桌機 / 1200px 以上


## v0.7.3 牌桌與手牌距離

- 手機 DOM 顯示順序：牌桌／上一手 → 我的手牌 → 操作。
- 上一手牌面置中，最多五張牌時不需要橫向捲動。
- 玩家狀態使用橫向滑動，避免四張狀態卡垂直堆疊。
- 牌局專注模式座位卡採雙欄，縮短進入牌桌前的距離。
- 操作面板黏附於手機底部，但需保留手牌底部空間，避免遮住最後一列。
- 320px、360px、390px、430px 寬度都應能看清牌桌、手牌與主要按鈕。

## v0.7.3 操作面板精簡

- 遊戲中預設只顯示選牌狀態、出牌與 Pass。
- 「推薦出牌」、「最小可出」、「清除選牌」、牌型建議及 Pass 設定預設收合。
- 點選「出牌輔助與 Pass 設定」後才展開輔助功能。
- 輔助區收合時，不得占用隱藏內容的版面高度。
- 320～430px 寬度下，出牌與 Pass 維持雙欄且觸控高度至少約 48px。
- 展開輔助區後，三個輔助按鈕不得超出面板；清除選牌在手機上獨占一列。
