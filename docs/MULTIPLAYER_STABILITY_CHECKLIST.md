# v0.6.5 多人實戰穩定檢查清單

## 1. 回合同步

- 輪到誰，只有該座位的真人玩家可以按「出牌」或「Pass」。
- 出牌 / Pass 送出前會記錄目前 `gameId`、`currentTurnSeat` 與 `security.revision`。
- Firestore transaction 執行時會再次比對上述狀態；若牌局已被其他玩家更新，會拒絕舊動作。
- 三家 Pass 後，`lastPlay` 會清空，`passCount` 歸零，領出權回到最後出牌者。

## 2. 出牌 / Pass 防連點

- 按下「出牌」或「Pass」後，按鈕會進入同步鎖定狀態。
- Firebase transaction 完成前不能再次送出。
- 若送出失敗，畫面會保留目前同步狀態並顯示錯誤訊息。

## 3. 下一局穩定

- 房主按「下一局 / 重新洗牌」後，會建立新的 `gameId`。
- 新局會重新洗牌、重新發牌、重新指定起手座位。
- 上一局的 `lastPlay`、`passCount`、`winnerSeat`、`results` 不會殘留到新局。
- 同一房間的 `totalScores`、勝場與累計總排名會保留。

## 4. 斷線 / 重連

- 玩家逾時未回應會被標示離線。
- 牌局進行中離線的真人座位會暫時由 AI 接管。
- 同一匿名 UID 回來時，可重新取回自己的座位。
- 房主離線時，房主會轉移給下一位已連線真人玩家。

## 5. 偵錯面板

截圖回報問題時，請讓畫面包含「多人同步偵錯面板」。重點欄位：

- 房號
- 我的座位
- 房主座位
- 目前回合
- Revision
- LastAction
- GameId
- 最後同步
