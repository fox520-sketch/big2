# v0.6.6 遊戲體驗檢查清單

## 排行榜

- 顯示同一房間累計總分、勝場與局數。
- 一局結束後應更新上局排名與最後同步時間。
- 下一局開始後應保留累計分數，不可歸零。
- 同一 gameNo 的結果不可被重複累加。

## 最近房號

- 建立或加入房間後會寫入本機最近房號。
- 最近房號應顯示房號、房主、狀態與最近事件。
- 過舊的最近房號會自動忽略，避免列表過長。
- 清除最近房號後，列表應立即清空。

## 房間列表

- 刷新 Firestore 最近房間時，若 Rules 允許讀取，應列出最近更新房間。
- 進行中房間按鈕顯示「回房」。
- 若 Rules 不允許列表讀取，應顯示提示，但不影響手動輸入房號加入。

## 手機版

- 手機直向顯示橫向提示。
- 手牌可左右滑動。
- 操作區保持 sticky，方便出牌 / Pass。
- 房間控制按鈕與房間列表按鈕在小螢幕上為單欄大按鈕。
- 結果表格可左右滑動，不應擠壓到看不清楚。

## 音效與動畫

- 音效總開關可關閉全部音效。
- 選牌、出牌、Pass、發牌、勝利、錯誤提示音可分別開關。
- 音量調整應立即套用。
- 關閉動畫後，手牌與回合提示不應再明顯閃動。
- 開啟動畫時，輪到的玩家卡片應有淡光提示。

## 多人出牌穩定性

- 出牌 / Pass 交易完成前，按鈕應鎖定。
- 手機快速連點時，只能送出一次。
- 若牌局 revision 已更新，應提示重新確認最新回合。
- Firebase 心跳或房間列表更新不應清空已選手牌。
- AI 接管出牌時，房主瀏覽器應自動推進，不應卡住。
