# v0.7.4 正式上線實測修正清單

此版維持免 Cloud Functions、不需要 Blaze。重點是修正上傳 GitHub Pages 後，朋友實際用手機加入房間時最常遇到的問題。

## 1. GitHub Pages

- `index.html` 載入 `styles/base.css?v=0.7.4`。
- `index.html` 載入 `src/main.js?v=0.7.4`。
- 目的：降低瀏覽器或手機快取造成舊版 JS / CSS 殘留的機率。
- 上傳後若仍看到舊版，請按 Ctrl + F5，或用手機無痕模式重新開啟。

## 2. Firebase 建立房間

- 按「建立房間」後，按鈕會顯示「建立中...」並暫時鎖定。
- 建立成功後會立刻顯示房號、邀請連結與座位 1。
- 若出現權限錯誤，請重新貼上本版 `firestore.rules` 並 Publish。

## 3. 加入房間 / 密碼房

- 按「加入房間」後，按鈕會顯示「加入中...」並暫時鎖定。
- 如果房間有密碼但未輸入或輸入錯誤，畫面會聚焦到房間密碼欄位。
- 如果房號不存在，會提示確認 6 碼房號或請房主重新複製邀請連結。

## 4. QR Code

- 按「顯示 QR Code」後會顯示載入狀態。
- QR 圖片載入成功後會顯示掃描說明。
- QR 圖片服務失敗時，會提示改用複製邀請連結，並提供備援圖片連結。

## 5. 多人同步

- 出牌 / Pass 維持 transaction、revision、gameId、currentTurnSeat 檢查。
- 多人出牌中會鎖定操作，避免手機連點造成重複送出。
- 出牌失敗時保留選牌與白話錯誤訊息。

## 6. 下一局與排行榜

- 房主按「下一局 / 重新洗牌」時會暫時鎖定按鈕。
- 下一局會建立新的 `gameId`。
- 排行榜以 `updatedGameNo` 防止同一局重複累加。

## 7. 手機測試建議

- 用一台電腦建立房間。
- 用手機掃 QR Code 或開邀請連結加入。
- 測試密碼房加入。
- 測試補 AI 空位。
- 測試一局完整打完與下一局。
- 測試手機鎖螢幕後重新回到網頁。
