# Firestore Rules 安全檢查（v1.0.0）

## 已強化
- 所有讀寫都要求 Firebase Authentication。
- 建房者 UID 必須等於 `hostUid`，且建房者已存在座位內。
- 更新既有房間時，要求呼叫者原本就是房內成員。
- 新玩家只能在 `waiting` 或 `finished` 狀態加入，而且只能新增／取代一個座位；房主、規則、密碼雜湊、局數及 game 不得一併更改。
- 只有房主可刪除房間。
- 限制房間最外層允許欄位。

## 無法由純 Rules 完整處理
- 驗證洗牌是否公平。
- 隱藏其他玩家手牌。
- 完整驗證出牌、Pass、AI 與計分演算法。
- 防止房內惡意成員竄改整個 game。

若需要正式競技級防作弊，必須將遊戲狀態改為可信任後端管理。
