# v0.7.5 手機實戰回歸與效能穩定檢查清單

## 裝置與版面

- Android Chrome：直向與橫向切換後，牌桌、手牌及操作區維持正確順序。
- iPhone Safari：網址列展開或收合時，頁面使用 `visualViewport` 更新高度，不因固定 `100vh` 產生大幅跳動。
- 平板：牌桌保持單欄清楚排列，手牌可水平滑動。
- 桌機：牌桌與操作維持雙欄，手牌完整顯示。
- 全部裝置：瀏海、Home Indicator 與圓角螢幕使用 safe-area 留白。

## 手牌與操作

- 最右側手牌後方保留滑動空間，最後一張牌可完整移到可視範圍。
- 手牌區使用水平觸控手勢，不影響頁面垂直捲動。
- 手機直向的操作面板改回正常文件流，緊接手牌但不覆蓋牌面。
- 手機橫向才使用側邊黏附操作面板，並受動態視窗高度限制。

## 網路與重連

- 離線時顯示「網路已中斷」，單人遊戲仍可使用。
- 網路恢復、頁面重新回到前景時，主動恢復 Firebase 心跳。
- 房主恢復連線後執行一次離線座位整理。
- 偵錯資訊包含 online 狀態與 visual viewport 高度。

## Firebase 效能

- 心跳間隔調整為 25 秒，離線判定放寬為 90 秒，降低手機背景切換的誤判。
- 只有房主執行離線座位與房主轉移整理。
- 房間快照忽略只有 `lastSeen`、`presenceUpdatedAt` 等心跳時間變動的更新，避免整個 UI 重繪。
- 同一房間重複呼叫監聽時沿用既有 listener，不再建立重複監聽。
- 可由房主手動清理自己建立、超過 24 小時且未進行中的房間。

## 下一局

- 房主不能在牌局進行中重複洗牌。
- 開始下一局會比對 `expectedGameNo`，避免多視窗或舊畫面重複建立新局。
- 下一局保留累計總分與勝場，但建立新的 gameId、手牌、上一手及 Pass 狀態。
